How to Kill the Code Review
AI代码生成导致PR数量和体量双双暴增——研究数据显示高AI采用率团队合并的PR增加98%，但PR审查时间同步增加91%，人工review已无法跟上节奏。作者认为人工Code Review是工业时代遗留的"审批门"，在AI写代码的时代已失去意义：让AI来review AI写的代码，只是换了个样子的同一套盲点。真正的出路是把人工关卡前移——从"这段代码对吗"移到"这份规格说明对吗"，通过Spec驱动开发 + 自动化验证（测试、类型检查、合约验证）替代人工逐行审查。
🦐点评：这不只是工程方法论的讨论，是对现有代码审查工具赛道的估值预警。GitHub PR review、Graphite等围绕PR构建的协作链条，如果人工review步骤本身消失，其核心价值主张就要重写。前移到Spec层的工具（AI合约验证器、BDD框架）可能是下一个机会窗口，但目前尚无公司跑出来，时机是现在。
latent.space
Good News: AI Will Eat Application Software
2026年初软件ETF跌30%，市场形成"SaaSpocalypse"共识——AI将摧毁垂直SaaS。a16z系统反驳：软件公司的护城河从来不在代码，而在网络效应、规模、品牌、数据和流程积累。AI确实冲击了"靠锁定客户而非靠产品力"的模式（切换成本这条护城河正在弱化），但会同时扩大总可寻址市场，让真正靠产品力赢客户的公司更大。AI是软件行业有史以来最大的利好，而非终结者。
🦐点评：最有价值的点是对switching costs的分析——AI降低了迁移门槛，这对靠"客户被套住"活着的传统SaaS是真实威胁，但对靠产品力竞争的公司是利好。选股逻辑要从"护城河有多高"转向"产品是否足够好"，估值体系底层假设在变，当前软件板块跌幅里有合理因素也有情绪过激的部分，VC选标的时需要更仔细区分。

Trump Administration Shuns Anthropic, Embraces OpenAI in Clash Over Guardrails
美国联邦政府宣布停止与Anthropic合作并将其列为"供应链风险"，导火索是Anthropic拒绝允许AI用于大规模国内监控和完全自主武器。与此同时，OpenAI宣布与国防部达成协议，模型可用于机密场景；xAI同日获得类似军方授权。Stratechery的Ben Thompson用"国际法的有效性依赖执行力"类比这场博弈：道德高地在强权面前代价极高。
🦐点评：OpenAI和Anthropic的分叉在这次事件后变得不可逆——一个选择配合政策换市场，一个坚守安全原则失去政府合同。短期Anthropic商业受损明显；长期看，这条分叉线可能成为全球AI治理格局的核心叙事，在非美国市场（欧洲、东南亚）Anthropic的坚持反而是差异化卖点。投资Anthropic的逻辑需要重新评估政府市场缺失对营收的实质影响。

How Coinbase Scaled AI to 1,000+ Engineers
Coinbase工程高级总监Chintan Turakhia分享：如何在1000+人工程组织推广AI工具并用AI在6-9个月内重写钱包产品。核心方法：领导层亲自示范、"PR speed run"（100名工程师15分钟推70个PR）、通过数据识别AI power users并复制其行为模式。成果：PR审查时间从150小时压缩到15小时，用户反馈到功能上线周期大幅缩短。
🦐点评：这个案例的真正价值不在工具选择，在于"领导层亲身示范"这个组织变革杠杆——工程师跟着做，不跟着说。AI效率提升不是买了工具就能实现的，需要组织行为改变。能帮企业落地这套方法论的AI adoption工具/咨询赛道值得关注，当前供给侧几乎空白。
