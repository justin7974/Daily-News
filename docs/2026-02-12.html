Harness engineering: leveraging Codex in an agent-first world
OpenAI 团队过去五个月进行了一个实验：用 Codex agent 从零构建一个完整的产品，全程 0 行手写代码。该产品已有内部用户和外部 alpha 测试者，涵盖应用逻辑、测试、CI 配置、文档、可观测性工具等。团队估计这种模式比手动写代码快约 10 倍，五个月内产出约百万行代码。人类工程师的工作转变为设计环境、指定意图、构建反馈循环，通过 prompt 与系统交互完成开发。
🦐点评：这是 AI 工程化的里程碑。关键洞察是稀缺资源从「编码能力」变为「问题分解能力」——人类不再写代码，而是学会如何向 agent 正确地提问。
openai.com
Claude Opus 4.6 vs. GPT-5.3 Codex: How I shipped 93,000 lines of code in 5 days
Lenny Rachitsky 分享了使用 Opus 4.6 和 GPT-5.3 Codex 两种模型在 5 天内编写 93,000 行代码的对比体验，探讨两者在代码生成任务上的差异。
🦐点评：开发者社区终于有了高质量的模型对比实测。Codex 作为垂直模型在代码任务上的表现值得关注。

Spotify Earnings, Individualized Networks, AI and Aggregation
Ben Thompson 分析 Spotify 最新财报，探讨 AI 对内容分发网络的影响。Spotify 作为内容聚合平台的本质意味着 AI 是维持性技术而非颠覆性技术。
🦐点评：Ben 的洞察很清晰——Spotify 的护城河在于内容网络效应，AI 是增强而非颠覆。但长期来看，「个性化网络」的概念值得深思。
