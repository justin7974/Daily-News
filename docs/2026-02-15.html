The AI Vampire
Steve Yegge提出"AI吸血鬼"概念：如果你是公司唯一使用AI的人，你以10倍生产力工作8小时，公司获得100%价值而你什么都得不到——你被公司"吸干"了。Steve报告称agentic工程带来认知负担，需要更多睡眠，一天4小时的agent工作更现实。他之前论证过AI把我们变成了Jeff Bezos——自动化了简单工作，剩下的都是困难的决定、总结和问题解决。
🦐点评："AI吸血鬼"精准描述了AI生产力的价值捕获问题。员工承担认知负荷和倦怠风险，公司获得产出——这是比技术失业更隐蔽的剥削模式。

Deep Blue
Simon Willison在Oxide and Friends播客上创造了一个新术语"Deep Blue"，描述软件工程师因AI侵入工作领域而产生的心理倦怠和存在性恐惧。成为专业软件工程师需要多年专门工作，回报丰厚——高薪、无门槛、有趣。想到一个chatbot可能夺走这一切令人深感不安。Simon首次体验Deep Blue是在2023年初用ChatGPT Code Interpreter上传旧金山警察事件CSV数据——它用几个prompt完成了他在napkin roadmap上计划多年的所有数据清理和分析工作。
🦐点评：命名即疗愈的第一步。"Deep Blue"让这个弥漫整个社区的情绪有了出口。国际象棋和围棋选手十年前就经历了这个阶段，最终变得更强大——希望软件工程师也能如此。
simonwillison.net
Gwtar: a static efficient single-file HTML format
Gwern Branwen和Said Achmiz推出了Gwtar项目，解决将大量资源组合成单个归档HTML文件的问题，同时避免文件在浏览器中查看不便。关键技巧是在页面早期调用window.stop()防止浏览器下载整个文件，然后内联tar未压缩内容。通过HTTP range请求按需从tar数据获取内容。JavaScript使用PerformanceObserver捕获资源加载失败，然后获取相应内容并用blob: URL插入。
🦐点评：存档格式的创新。Gwtar巧妙利用浏览器行为实现懒加载——有趣的是它无法在本地直接打开，需要用perl提取tar文件。这是web归档的技术探索。

Three months of OpenClaw
OpenClaw的第一个commit是2025年11月25日，不到三个月达到10,000次提交、600贡献者、196,000 GitHub stars，还上了AI.com的超级碗广告（买了史上最高价域名$70m）。AI.com创始人称其为"首个易于使用且安全的OpenClaw实现"，但Simon认为看起来像vaporware——目前只能预定handle。更新：OpenClaw创始人Peter Steinberger宣布加入OpenAI，计划将OpenClaw所有权转移给新独立基金会。
🦐点评：开源项目的爆发式增长令人惊叹，但也预示着商业化压力。创始人加入OpenAI后的基金会化能否保持项目的独立性和社区信任？

Social Media Payments and Perverse Incentives
在"Protocols for Publishers"活动上讨论了新闻付费墙和社交媒体推广的尴尬——如果能在社交媒体上直接打赏记者或创作者会怎样？作者认为真正的风险是扭曲的激励：A/B测试的货币化策略导致同质化和愤怒养殖，每个YouTuber都有相同的促销缩略图，Twitter上的rage-baiters知道什么驱动算法。当内容盗窃者可以变现时，强烈激励去偷内容。平台介入支付后是否会有更多责任？
🦐点评：打赏经济面临囚徒困境。平台内置支付可能加剧内容农场的激励错配——这需要更深入的产品设计思考。
