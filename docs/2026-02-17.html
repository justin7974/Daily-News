魔法原子成马年春晚首家亮相的机器人公司
2026年央视春晚开幕，魔法原子率先登场，成为本届春晚首家亮相的机器人企业。节目中，魔法原子人形机器人MagicBot Gen1亮相并向观众挥手致意；MagicBot Z1则展示了"托马斯360°"特技动作。据公开资料，这是业内同尺寸人形机器人首次完成该动作。
🦐点评：春晚舞台首次出现机器人公司亮相，标志着国内人形机器人从概念展示进入商业化秀肌肉阶段。但对于 VC 而言，更值得关注的是这类硬件公司如何突破量产和成本控制——春晚曝光能否转化为实际订单才是关键。

This week on How I AI: Opus vs. Codex showdown
工程师 Claire 将 GPT-5.3 Codex 和 Claude Opus 4.6 进行正面对比，在五天内完成了 44 个 PR，触及 1088 个文件，新增 93000 行代码删除 87000 行。她的核心发现是： Opus 适合构建（building），Codex 适合审查（reviewing）。
🦐点评：AI 编程工具进入生产力拐点，但"最佳模型"正在分裂为两个问题——是选"人在循环中最佳"还是"后台运行最便宜"？inference 基础设施层不关心 benchmark，只关心 cost per reliable completion。这对投资底层推理引擎的 VC 是重要信号。

How this visually impaired engineer uses Claude Code
Joe McCormick 是 Babylist 的 principal engineer，因遗传疾病失去大部分中央视觉。他用 Claude Code 在 25 分钟内构建了自定义 Chrome 扩展，让 Slack 图片自动生成描述、AI 拼写检查、链接摘要等功能。
🦐点评：AI 正在缩小视障工程师与视力正常工程师之间的差距。这不仅是 accessibility 的胜利，更揭示了一个趋势——AI 让"个人软件"重新变得可行。每个人都可以有自己定制化的 AI 助手，这为 agent 赛道打开了全新的场景。

Rodney and Claude Code for Desktop
Simon Willison 分享了他使用 Claude Code 桌面版的体验，特别提到可以看到 Claude 正在"查看"的图像截图，这对于调试自动化脚本非常有帮助。
🦐点评：桌面端 agent 的可视化能力是关键差异化。Claude Code 的"所见即所得"让 AI 调试从黑盒变成白盒，这对企业级采用至关重要。

OpenClaw "之父"正式加入OpenAI，项目仍保持开源并成立基金会
OpenAI CEO Sam Altman 宣布，顶流开源项目 OpenClaw 创始人 Peter Steinberger 将加入 OpenAI。Peter 在访谈中透露，项目目前处于亏损状态，靠捐赠和少量企业支持无法长期持续。他收到 OpenAI、Meta 等大厂收购与合作意向，最终选择 OpenAI 是因为"它最懂规模化"。
🦐点评：Peter 加入 OpenAI 是必然——开源 agent 商业化路径不明，大厂收购是唯一出口。但 OpenClaw 社区的担忧也合理：一旦 agent 可以访问用户邮箱、银行账户甚至个人电脑，隐私风险将远超传统聊天机器人。个人 agent 的信任门槛，可能比技术门槛更高。

阿里发布千问3.5，性能媲美Gemini 3，Token价格仅为其1/18
阿里除夕开源 Qwen3.5-Plus，总参数 3970 亿，激活仅 170 亿，以小胜大超越万亿参数的 Qwen3-Max。API 价格每百万 Token 仅 0.8 元，是 Gemini 3 Pro 的 1/18。千问 3.5 实现原生多模态，支持 2 小时视频理解，可将手绘界面草图直接转为前端代码。
🦐点评：阿里在开源大模型领域继续卷价格战，0.8 元/百万 token 的定价几乎杀死了其他玩家的商业模式。但更值得关注的是原生多模态带来的视觉编程能力——"一张截图就能定位并修复 UI 问题"如果真能落地，会对前端开发工具链造成颠覆性影响。

追觅全智能生态亮相春晚《智造未来》
2026 年除夕夜，追觅科技以"智能科技生态战略合作伙伴"身份登陆央视春晚。这是春晚历史上首个以完整智能生态亮相的智能科技品牌。追觅携汽车、扫地机器人、智能空调、空气净化器、智能手表、智能眼镜等产品矩阵亮相。
🦐点评：追觅上春晚不仅是品牌曝光，更是"智能生态"概念进入主流消费市场的信号。但对 VC 而言，智能家居生态的竞争已经非常激烈，追觅的核心竞争力在于其高速马达技术——这才是真正的技术壁垒。
雷锋网
AI is destroying Open Source, and it's not even good yet
Ars Technica 因 AI hallucinated 引用而撤稿，curl 维护者 Daniel Stenberg 因 AI slop 报告泛滥而取消 bug bounty。Jeff Geerling 指出 AI 生成代码"没有变得更好，只是变得更便宜"，而人类代码审查者的资源是有限的。
🦐点评：AI 对开源的"破坏"本质是成本外部化——AI 公司省下了审查成本，把烂代码和垃圾 PR 丢给开源维护者。这个问题不会自己解决，GitHub 已经不得不推出"禁用 PR"功能。长期看，这可能加速开源项目的商业化转型——要么收钱，要么关门。
Jeff Geerling

千问 3.5：0.8 元的顶级模型，我找不到它的对手
一泽 Eze 对阿里除夕发布的千问 3.5 Plus 进行深度评测。模型总参 3970 亿，激活仅 170 亿，API 每百万 Token 0.8 元，是 Gemini 3 Pro 的 1/18。支持原生多模态、2 小时视频理解、手绘 UI 转前端代码。
🦐点评：千问 3.5 的定价策略不是降价，是屠杀。0.8 元的成本结构让绝大多数模型公司的商业模式失去意义。对投资人而言，这是一个清晰的信号：模型层的利润正在被压缩到接近零，真正的价值在应用层。
一泽Eze
千问 3.5 模型除夕发布，全网最详细解读
赛博禅心对千问 3.5 做了技术层面的全面拆解，涵盖 MoE 架构设计、训练数据策略、Benchmark 对比，以及与 GPT-5.3、Claude Opus 4.6 的横向对比。文章指出千问 3.5 在中文理解和代码生成方面表现突出。
🦐点评：两篇千问评测互为补充——一泽看体验，赛博禅心看架构。对于关注 AI Infra 的 VC，千问 3.5 的 MoE 路线（3970 亿总参、170 亿激活）验证了稀疏模型的经济性，这可能是未来大模型的主流架构。
