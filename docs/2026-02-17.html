OpenClaw "之父"正式加入OpenAI，项目仍保持开源并成立基金会
Sam Altman宣布OpenClaw创始人Peter Steinberger加入OpenAI，致力于推动下一代个人智能体研发。Peter透露项目目前处于亏损状态，爆红后收到OpenAI、Meta等大厂收购与合作意向，最终选择OpenAI是因为"它最懂规模化，能把技术安全地推广给更多人"。OpenClaw将转为基金会，保持开源和独立。
🦐点评：Peter加入OpenAI而非Anthropic，某种程度上承认了开源Agent商业化路径的困境——单纯靠捐赠无法持续。OpenClaw基金会化后能否保持社区活力有待观察，但"让Agent进入每个人电脑"的愿景与OpenAI的消费者战略高度契合。
infoq.cn
How this visually impaired engineer uses Claude Code to make his life more accessible
Joe McCormick是Babylist principal engineer，因罕见遗传病失去大部分中心视觉。他用Claude Code在25分钟内构建自定义Chrome扩展，让Slack图片自动描述、即时修正拼写错误、一键摘要链接。AI使他能够"做以前不可能做的事"——给孩子们读书。
🦐点评：AI编程工具正在缩小视障工程师与健全工程师之间的能力差距。这不仅是 accessibility 故事，更是"个人软件"复兴的信号——LLM让个人开发者能快速构建满足自身需求的工具，这将催生新一轮长尾创新。
lennysnewsletter.com
This week on How I AI: AI for Accessibility — and the Opus vs. Codex Showdown
Claire对比测试GPT-5.3 Codex与Claude Opus 4.6：五天提交44个PR、修改1088个文件、增减9.3万行代码。核心洞察是"不是Opus或Codex，而是Opus构建+Codexreview"——Opus像"急于求成的产品工程师"，适合创意 greenfield 工作；Codex像"不会自己动手的Principal Engineer"，擅长review和找边界情况。
🦐点评：AI编程工具进入"分工协作"阶段。两个模型组合的效率远超单一模型，这对IDE/Coding Agent的产品设计有重要启示——工具链的价值可能大于单点模型能力。
lennysnewsletter.com
AI is destroying Open Source, and it's not even good yet
Ars Technica因AI hallucinated quotes撤稿，curl maintainer因AI slop bug report占比从15%降到5%而取消bug bounty，GitHub新增"关闭PR"功能。Jeff Geerling警告：AI生成的代码越来越容易，但不会越来越聪明；人类reviewer的资源是有限的。"如果这已是问题，OpenClaw的发布和OpenAI招聘Peter只会让情况更糟"。
🦐点评：Jeff的判断偏悲观，但核心问题真实——开源维护者正在被AI生成的低质量PR淹没。不过换一个角度：能用AI批量生产PR的团队，说明代码库本身足够标准化，这恰恰是投资时看重的"工程化能力"信号。
jeffgeerling.com
Rodney and Claude Code for Desktop
Simon Willison演示用Rodney（他设计的CLI浏览器自动化工具）配合Claude Code Desktop：可以通过截图实时看到Agent正在操作什么，无需等待代码推送到GitHub才能验证效果。关键是Rodney的--help输出包含Agent需要的所有信息。
🦐点评：Agent的可观测性是工程化落地的关键。Simon的实践揭示了一个趋势——未来的Agent工具不仅要"能干活"，还要"能让人看懂在干什么"。
simonwillison.net

千问 3.5：0.8 元的顶级模型，我找不到它的对手
阿里除夕发布Qwen3.5-Plus，3970亿参数激活170亿，性能媲美Gemini 3 Pro，API价格每百万Token仅0.8元，是Gemini 3 Pro的1/18。文章详细对比了与前代Qwen3-Max的性能差异，以及在推理、编程、Agent智能体等基准上的表现。
🦐点评：阿里用"小模型打败大模型"的策略——以不到40%的参数量获得超万亿参数模型的顶尖性能。这对国内大模型竞争格局影响深远：价格战已从"卷Token"升级到"卷有效推理成本"，中小厂商的生存空间被进一步压缩。
一泽Eze
千问 3.5 模型除夕发布，全网最详细解读
全网最详细的Qwen3.5技术解读，涵盖预训练策略、视觉文本混合token、多语言数据增强等核心技术细节。文章指出Qwen3.5实现了从纯文本模型到原生多模态模型的代际跃迁。
🦐点评：Qwen3.5的真正壁垒不在模型架构，而在阿里多年积累的中文互联网数据资产。当"眼睛"看到世界后，模型的推理和知识密度大幅提升——这验证了多模态是通向AGI的必经之路。
赛博禅心

Peter Steinberger接受Lex Fridman深度访谈，讲述OpenClaw爆红后的经历：被Anthropic追着要求改名、加密社区恶意骚扰、账号抢注。最终选择加入OpenAI是因为"它最懂规模化，能把技术安全地推广给更多人"。
🦐点评：这篇没全文，但Peter的核心洞察值得注意——他强调Agent需要"权限边界"设计，用户对"Agent访问邮箱/银行账户"的担忧是真实的市场准入门槛。
