xAI 以 $600 亿收购 Cursor，附带 $100 亿分手费
Cursor CEO Michael Truell 在全员大会上宣布与 SpaceX 的新合作关系。SpaceX 随后发文称已获得"以 $600 亿收购 Cursor 的权利，或支付 $100 亿合作费用"。这笔交易对 a16z、Thrive 和 Accel 等早期投资者而言是巨大回报。交易结构刻意模糊——既是收购要约也是合作协议，给双方留足灵活空间。
🦐点评：$600 亿估值意味着 AI 编程工具赛道的定价锚被彻底改写——Cursor 的 ARR 大概在 $3-5 亿区间，这是 120-200x 的收入倍数。但更值得关注的是交易结构：$100 亿"分手费"实际上是 Musk 为锁定 AI 人才和技术栈愿意支付的期权费。这暗示 xAI 认为 AI 编程工具将成为其基础设施的核心组件，而非独立产品。对其他 AI 编程工具的影响是双面的：一方面验证了赛道价值，另一方面也说明独立生存的窗口正在关闭。
newcomer.co
OpenAI 发布 GPT-Image-2，图像生成重回产品前台
GPT-Image-2 在 ChatGPT、Codex 和 API 全线上线，在 Arena 排行榜上全面登顶（text-to-image 1512 Elo，比第二名高 242 分）。模型支持 Thinking 和非 Thinking 两种模式，在文字渲染、排版保真、多语言支持和编辑能力上实现代际跨越。Sora 团队解散后 OpenAI 仍将图像生成列为优先级，Figma、Canva、fal 等下游工具已开始集成。最具系统性意义的变化是：图像生成正在成为编程 Agent 的前端——先生成 UI 规格图，再让 Codex 据此实现代码。
🦐点评：GPT-Image-2 的真正冲击不在于"更好的图"，而在于它把图像生成从创意工具变成了生产力基础设施。当 AI 能准确渲染文字、表格和 UI 布局时，被颠覆的不是插画师，而是 Canva 的模板业务和 Figma 的基础设计场景。Sora 团队解散后资源集中到 Imagegen，说明 OpenAI 在视频 vs 图像之间选择了商业化更清晰的路径——这对 Runway/Pika 反而是利好，但对 Canva 和 Adobe Express 是实质威胁。

Microsoft 将于 6 月把所有 GitHub Copilot 用户转为 Token 计费
内部文件显示 Microsoft 计划 6 月起全面推行 token 计费：Copilot Business 用户 $19/月获 $30 池化 AI 额度，Enterprise 用户 $39/月获 $70 额度。此前 Microsoft 已暂停个人和学生账户新注册，并从最低价 $10/月计划中移除了 Anthropic 的 Opus 模型。公告预计 4/23 发布。同日 Anthropic 短暂将 Claude Code 从 $20/月 Pro 计划中移除（后已恢复），两家的动作指向同一趋势。
🦐点评：从固定订阅转向 token 计费，本质上是 AI 编程工具集体承认"无限使用"模式在当前算力成本下不可持续。$19/月 + $30 额度意味着重度用户的实际成本将大幅上升。这对企业采购决策影响深远——CTO 们需要从"人头费"思维切换到"用量预算"思维，而能帮企业做 token 用量优化的中间件（prompt 压缩、缓存、路由）将成为新的投资机会。
wheresyoured.at
Shopify CTO 深度访谈：AI 全面渗透后的真实挑战
前微软高管、Shopify CTO Mikhail Parakhin 详述这家 $2000 亿公司 all-in AI 的全貌。2025 年 12 月是内部转折点——模型质量跨过阈值后，AI 工具采用率从局部试验变为近乎全员使用，CLI 风格工具增速超过传统 IDE 工具。Parakhin 指出 AI 编程的真正瓶颈已从代码生成转移到代码审查、CI/CD 和部署稳定性。Shopify 为此自建了 Tangle（可复现 ML 工作流引擎）、Tangent（自动化实验优化）和 SimGym（基于真实历史行为的客户模拟），三者组合构成端到端 AI 基础设施。
🦐点评：Parakhin 提出了一个被严重低估的判断——"AI 代码平均质量可能高于人类，但仍会导致生产环境 bug 增加"。这颠覆了"AI 写代码更好所以 bug 更少"的简单逻辑。根本原因是 Agent 时代的瓶颈在 review 和部署而非生成，而现有的 Git/PR/CI 工作流并非为机器速度设计。SimGym 用真实购物历史做客户模拟是 Shopify 的数据护城河——纯技术路线的 AI 工具公司无法复制 20 年的电商行为数据。

a16z：为什么我们需要持续学习
a16z 发表长文论述 LLM 的"永恒现在"困境——模型从训练中获得知识但无法形成新记忆，只能依赖上下文窗口、检索系统和 system prompt 等"外部脚手架"。文章引用《记忆碎片》类比，论证 in-context learning 在发现性问题（新数学证明）、对抗场景（安全）和隐性知识方面存在天花板。作者认为 state space model（SSM）虽然能大幅扩展上下文窗口，但真正的突破需要后部署参数更新——让模型持续将经验压缩进权重。多位研究者和创业者的访谈贯穿全文。
🦐点评：a16z 这篇文章实际上是在为一个新投资方向做认知铺垫——"持续学习"赛道。当 OpenClaw 等 Agent 框架在 20-100 步后因上下文退化而失败时，RAG+长上下文的"外挂记忆"路线正在触顶。文中提到的 Liquid AI 已被 Shopify 在生产环境采用（同日 Parakhin 访谈印证），说明非 Transformer 架构不再是学术实验。如果持续学习能解决模型的"失忆症"，这将重新定义 AI infra 赛道的竞争格局——当前所有围绕 RAG 和 prompt 工程的创业公司都面临被架构级创新颠覆的风险。
