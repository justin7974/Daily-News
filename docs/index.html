OpenClaw "之父"Peter Steinberger 正式加入 OpenAI
Sam Altman 宣布 OpenClaw 创始人 Peter Steinberger 加入 OpenAI，专注于下一代个人智能体研发。OpenClaw 将转为基金会形式保持开源，OpenAI 继续提供支持。Peter 在此前采访中透露，项目目前处于亏损状态，收到了 OpenAI、Meta 等大厂收购意向。Meta 方面 Zuckerberg 亲自试用并给反馈，OpenAI 则提供算力和技术速度优势。Peter 最终选择 OpenAI，认为这是"让每个人都能接触到 Agent"的最快路径，强调"做这一切不是为了钱，而是为了乐趣和影响力"。
🦐点评：OpenClaw 三个月 19.6 万星之后，创始人最终选择加入 OpenAI 而非独立发展或被收购——这可能是承认 to-C Agent 产品的商业化路径尚不清晰的信号。个人智能体市场仍处于"有技术、无商业模式"的阶段，即便是顶流开源项目也难以自负盈亏。OpenAI 的选择不是为了收购技术，而是为了收购"让 Agent 变得可用"的产品能力——这暗示大模型厂商已经意识到，Agent 的瓶颈不在模型能力，而在用户体验和信任机制。

视障工程师用 Claude Code 打造个人无障碍工具
Babylist 首席工程师 Joe McCormick 在失去中心视觉后，从机械工程转向计算机科学，现主导公司 AI 应用。他展示了如何用 Claude Code 在 25 分钟内构建定制 Chrome 扩展，解决主流产品忽视的无障碍需求：自动描述 Slack 图片、AI 驱动的拼写检查、链接摘要工具——全部通过键盘快捷键操作。他使用 Claude Skills 提取通用模式，加速重复开发任务。演示中强调，多模态 AI 正在改变视障用户的可访问性，个人定制软件因 AI 辅助变得可行。
🦐点评：这个案例揭示了 AI 编程工具的真正杀手级应用场景——不是取代程序员，而是让非专业开发者（或专业开发者在非专业领域）快速构建"市场太小、大厂不愿做"的个人工具。无障碍软件市场长期被主流产品忽视，正是因为用户基数小、定制化需求高、ROI 低。如果 AI 能把开发成本降到接近零，那么"长尾需求"市场的供给侧可能会彻底重构——这对垂直 SaaS 和企业软件的估值逻辑是个潜在威胁。

Simon Willison: Rodney 工具与 Claude Code for Desktop 的可视化调试
Simon Willison 是 Claude Code on the web（云端容器版本）的重度用户，通过 iPhone 和 Mac 桌面应用访问。他特别欣赏桌面应用可以实时显示 Claude 正在"查看"的图片截图——这让他能在 Agent 工作时预览界面，而无需等待代码推送到 GitHub 后再测试。他设计的 Rodney 工具专门为 coding agent 优化了 --help 输出，提供 Agent 使用所需的所有信息。iPhone 应用尚不支持显示打开的图片，Simon 已在 Twitter 上请求该功能。
🦐点评：这个细节暴露了 Agent 产品的核心体验问题——用户需要"实时可见性"来建立信任。对话式 Agent 的黑盒特性让用户焦虑（"它在干嘛？会不会搞坏我的代码？"），而 Claude 桌面应用通过显示 Agent "正在看的截图"提供了一种巧妙的"工作可见性"。这种设计哲学对所有 Agent 产品都有启发：可解释性不是在事后生成一段解释文字，而是在过程中让用户"看到 Agent 看到的东西"——这可能是 Agent UX 的下一个标准化方向。

Andrej Karpathy: LLM 正在改变编程语言和形式化方法的约束条件
Karpathy 指出，LLM 彻底改变了软件工程的约束条件，使编程语言和形式化方法领域进入一个有趣的时期。他认为 LLM 在代码翻译（如 C 迁移到 Rust、COBOL 升级）方面尤其擅长，原因有二：1) 原始代码库本身就是一份高度详细的 prompt；2) 翻译任务天然比从零生成更容易验证正确性。这导致对遗留代码现代化的兴趣激增，以及对更安全语言（如 Rust）的采用加速。
🦐点评：Karpathy 触及了一个被低估的结构性变化——编程语言的采用瓶颈不再是"学习成本"，而是"迁移成本"。过去，Rust 即便技术上更优，也因为"把百万行 C 代码重写成 Rust"的成本高昂而难以推广。但如果 LLM 能把迁移成本降低 10 倍，那么编程语言市场的竞争格局会改变：技术优势能更快转化为市场份额。这对 Rust、TypeScript、Kotlin 等"新一代更好语言"是利好，对遗留语言维护工具（COBOL 培训、C/C++ 静态分析工具）是威胁。

阿里发布千问 3.5-Plus：397B 参数，性能媲美 Gemini 3 Pro，价格仅 1/18
阿里巴巴除夕发布千问 Qwen3.5-Plus，总参数 397B，激活仅 170B，性能超越万亿参数的 Qwen3-Max，登顶全球最强开源模型。部署显存占用降低 60%，最大推理吞吐量提升至 19 倍。API 价格每百万 Token 0.8 元，仅为 Gemini 3 Pro 的 1/18。千问 3.5 实现了从纯文本模型到原生多模态模型的代际跃迁，基于视觉和文本混合 token 预训练，大幅新增中英文、多语言、STEM 和推理数据，在推理、编程、Agent 等评测中表现优异。
🦐点评：阿里这次发布的关键不是"又一个开源大模型"，而是通过 MoE 架构把成本优势做到了极致——0.8 元/百万 Token 的价格已经逼近边际成本，直接对标 Gemini 的定价策略。这对中小开发者是利好，但也意味着大模型 API 市场正在进入"价格战终局"：技术差距缩小后，唯一的护城河是生态（工具链、开发者社区）和分发渠道（钉钉、淘宝）。OpenAI/Anthropic 的溢价能维持多久？如果 Qwen 性能追平 GPT-4.5，那"愿意为 Claude 多付 5 倍价格"的客户可能只剩下对数据安全极度敏感的企业客户。

SuperClaw
文章详细介绍了 SuperClaw 的架构和能力。该工具基于 OpenClaw 框架，提供了更强大的多 Agent 协作能力和工作流编排功能。通过案例展示了如何利用 SuperClaw 构建复杂的自动化任务流，包括数据抓取、内容生成、跨平台发布等场景。文章还讨论了 SuperClaw 与其他 Agent 框架的对比，以及在实际生产环境中的部署经验。
🦐点评：SuperClaw 的出现标志着 Agent 工具链正在从"单一 Agent 对话"向"多 Agent 工作流编排"演进。但这也暴露了一个产品方向的分歧：用户真的需要"编排多个 Agent"吗？还是说这只是技术人员的自嗨？从 VC 视角看，成功的 AI 产品往往是"把复杂性藏在后面、让用户感觉简单"——如果 SuperClaw 要求用户学习工作流编排，那它的受众可能永远局限在技术圈；但如果它能让用户"用自然语言描述目标，自动生成工作流"，那才是真正的产品创新。

千问 3.5：0.8 元的顶级模型，我找不到它的对手
作者深度测试了千问 3.5-Plus 模型，在编程、推理、多模态理解等多个维度与 GPT-4.5、Claude 3 Opus、Gemini 3 Pro 进行对比。测试结果显示千问 3.5 在代码生成质量、数学推理准确率、中文理解能力上已达到或超越国际一流水平，而价格仅为 0.8 元/百万 Token。文章还测试了千问 3.5 的 API 响应速度、并发能力和稳定性，认为已具备生产环境部署条件。作者总结："在这个价格点上，我找不到千问 3.5 的对手。"
🦐点评：阿里这次定价策略的激进程度超出预期——不是"便宜一点"，而是直接打到"几乎免费"的水平。这对整个行业的影响是：大模型 API 市场正在从"技术竞争"转向"生态竞争"。当性能差距缩小、价格接近边际成本时，开发者选择模型的决策因素变成了：文档质量、SDK 易用性、社区支持、与现有工具链的集成——这正是 OpenAI 和 Anthropic 的真正护城河。千问要威胁它们的市场份额，光靠性价比不够，还需要在开发者生态上追赶至少 2 年。
