🎙️ This week on How I AI: Opus vs. Codex showdown, and AI for accessibility
本周 How I AI 播客探讨两个主题：一是 GPT-5.3 Codex 与 Claude Opus 4.6 的正面对决，测试场景包括重构复杂组件和上线生产代码，结果显示 Opus 擅长构建（80%-90% 完成度），Codex 擅长 Code Review 发现边缘案例，最佳实践是 Opus 构建 + Codex 审核 + Opus 修复的组合 workflow；二是视障工程师 Joe McCormick 展示如何用 Claude Code 构建无障碍工具，包括 Slack 图片描述、即时拼写检查、链接摘要等功能。
🦐点评：Opus + Codex 的组合模式揭示了 AI 编码工具的分层价值——创意构建与质量审核是两种截然不同的能力。对 VC 而言，这意味着纯做 Code Review 的工具（如 Codex）可能面临价值被压缩的风险，而能同时提供构建+审核完整闭环的工具（如 Cursor/Cline）更具护城河。
lennysnewsletter.com
How this visually impaired engineer uses Claude Code to make his life more accessible
Babylist 工程师 Joe McCormick 失去中心视力后，利用 Claude Code 在 25 分钟内构建自定义 Chrome 扩展，解决日常无障碍需求：Slack 图片转文字描述、AI 拼写检查器、链接摘要生成器等。他展示了一套完整的工作流——用 Claude Skills 提取通用模式加速后续开发，键盘快捷键替代鼠标操作，配合屏幕阅读器实现高效开发。
🦐点评：AI 让"个人软件"重新变得可行——以前需要团队才能做的工具，现在个人用自然语言就能构建。这对 accessibility tech 赛道是结构性利好：需求一直存在，但供给侧的开发成本大幅下降。可能催生一批服务视障/听障人群的微型 SaaS。
lennysnewsletter.com
Rodney and Claude Code for Desktop
Simon Willison 展示了他如何结合 Rodney（自研 CLI 浏览器自动化工具）与 Claude Code 桌面端进行开发。通过桌面应用可以直接查看 Claude 正在"阅读"的图片截图，无需等待代码推送到 GitHub 就能实时预览效果。他用 uvx rodney click + screenshot + Read 的组合完成了 Datasette 菜单测试。
🦐点评：Rodney + Claude Code 的组合代表了"本地 Agent"的一个可行路径——不是把所有计算放到云端，而是让 Agent 能安全地操作本地资源（浏览器、终端）。对 Agent 工具开发者而言，本地执行能力可能是差异化竞争点。
simonwillison.net
AI is destroying Open Source, and it's not even good yet
Ars Technica 因 AI 幻觉引用而撤稿，curl 维护者因 AI slop 报告放弃 bug bounty，GitHub 新增禁用 PR 功能——Jeff Geerling 描述了 AI 给开源生态带来的乱象。AI 生成代码质量已触及天花板，但人类代码审查资源有限，AI slop PR 却源源不断涌入。他警告 OpenClaw 的发布和创始人加入 OpenAI 会让情况恶化，类比 crypto/NFT 泡沫。
🦐点评：AI Agent 的"民主化"与开源生态的"质量维护"之间存在结构性矛盾。GitHub 禁用 PR 是标志性事件——当工具变得太容易使用时，筛选成本反而上升。对投资人而言，这可能催生两类机会：1) AI 代码质量检测/过滤工具；2) 更稀缺的人类代码审查服务。
jeffgeerling.com
阿里发布千问3.5，性能媲美Gemini 3，Token价格仅为其1/18
阿里除夕开源 Qwen3.5-Plus，总参数 3970 亿、激活仅 170 亿，性能超越万亿参数的 Qwen3-Max。API 价格每百万 Token 仅 0.8 元，是 Gemini 3 Pro 的 1/18。实现原生多模态训练，在 MMLU-Pro、GPQA、IFBench 等基准超越 GPT-5.2 和 Claude 4.5。阿里云推理吞吐量最大提升 19 倍。千问 App 的 AI 购物 Agent 春节期间完成 1.2 亿笔订单。
🦐点评：阿里把开源大模型的价格打到地板价（0.8 元/百万 token），直接威胁所有中国大模型公司。千问 3.5 的性价比结合阿里电商场景的数据飞轮，可能让阿里在 AI 应用层形成显著壁垒。投资人需要重新评估国内大模型公司的护城河——纯模型层公司可能难以招架。
leiphone.com
