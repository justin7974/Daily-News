🔥 精选推荐

刚刚，OpenClaw "之父"正式加入OpenAI，项目仍保持开源并成立基金会
OpenAI CEO Sam Altman 宣布，顶流开源项目 OpenClaw 创始人 Peter Steinberger 将加入 OpenAI，致力于推动下一代个人智能体的研发。Peter 在访谈中透露，项目爆红后经历了加密社区恶意骚扰、账号抢注等困境，最终通过精密"作战式"操作完成更名。他表示项目目前处于亏损状态，靠捐赠和少量企业支持维系，爆红后收到 OpenAI、Meta 等大厂收购与合作意向，最终选择加入 OpenAI 是因为"这是实现改变世界目标最快的途径"。
🦐点评：OpenClaw 三个月 19.6 万星——但开源 Agent 的商业化路径仍然模糊，Peter 加入 OpenAI 可能是承认 to-C 变现走不通的信号。真正值得关注的不是技术本身，而是 Agent 产品的信任问题：权限边界、数据访问、审计机制如果无法回答，即便能力成熟也难以跨越用户信任门槛。
infoq.cn
This week on How I AI: Opus vs. Codex showdown, and AI for accessibility
Lenny's Podcast 推出两期重磅内容：一是 Claire 将 GPT-5.3 Codex 和 Claude Opus 4.6 进行正面 PK，五天内提交 44 个 PR、修改 1088 个文件、加删近 20 万行代码，她的洞察是"Opus 负责构建，Codex 负责 review"；二是视障工程师 Joe McCormick 展示如何用 Claude Code 开发无障碍 Chrome 扩展，让 AI 编程工具真正赋能残障开发者。
🦐点评：Opus 构建、Codex review 的分工模式揭示了 AI 编程的工作流范式——创意型任务和审查型任务需要不同模型。但更值得关注的是无障碍场景：AI 正在缩小视障工程师与健全工程师之间的能力差距，这是技术平权的真正落地，而非概念口号。
lennysnewsletter.com
How this visually impaired engineer uses Claude Code to make his life more accessible
Joe McCormick 是 Babylist 首席工程师，因罕见遗传病失去中心视觉。他演示了如何用 Claude Code 在 25 分钟内构建定制的 Chrome 扩展，包括描述 Slack 图片、AI 拼写检查、链接摘要等功能。他创建了 Claude Skill 来加速重复开发，让第五十个扩展的开发时间大幅缩短。
🦐点评：个人软件正在成为 AI 时代的新品类——主流产品忽略的个性化需求，个人可以用 AI 工具快速实现。这对 VC 意味着什么？关注"个人 AI 工具"赛道的投资机会，尤其是服务弱势群体的细分场景。
lennysnewsletter.com
Rodney and Claude Code for Desktop
Simon Willison 展示了他如何结合 Rodney（自研 CLI 浏览器自动化工具）和 Claude Code 桌面版实现可视化编程：运行测试时 Claude 可以实时查看截图，突破了纯文本交互的局限。他通过"运行测试→查看截图→反馈修正"的循环，实现了所见即所得的 AI 编程体验。
🦐点评：Simon 的工作流揭示了一个趋势——AI 编程工具正在从"纯代码生成"向"可视化调试"演进。关键洞察是：模型能"看到"屏幕比能"生成"代码更有价值，因为这解决了 AI 无法自我纠错的根本问题。
simonwillison.net
AI is destroying Open Source, and it's not even good yet
Jeff Geerling 指出 AI 正在从多个维度破坏开源生态：Ars Technica 因 AI hallucination 撤稿、curl 维护者因 AI slop 漏洞报告放弃 bug bounty、GitHub 新增功能允许禁用 PR。他将当前 AI 热潮比作 Crypto/NFT 泡沫，警示"AI 公司摧毁的东西比创造的多"。
🦐点评：Jeff 的核心论点是 AI 公司正在透支开源社区的信任资本——用开源数据训练模型、商业化变现，却让维护者承担质量审查成本。这个逻辑对 VC 的启示：投资 AI 基础设施（如代码审查工具、自动化测试）比投资模型本身更有长期价值，因为审查需求会爆发式增长。
jeffgeerling.com
