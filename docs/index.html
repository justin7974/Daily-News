Amazon Plans Smartphone Comeback More Than a Decade After Fire Phone Flop
Reuters 独家报道，Amazon 正在内部代号"Transformer"项目下重启智能手机计划，由设备与服务部门开发。据四位知情人士透露，这款手机定位为"移动个性化设备"，可基于用户购买历史和偏好做推荐，整合 Alexa+ AI 助手。距离 2014 年 Fire Phone 惨败已过去十年以上，Amazon 试图在 AI 时代重新切入移动入口。
🦐点评：Amazon 重返手机赛道的真正意图不是硬件本身——Fire Phone 证明了 Amazon 做不好消费电子。关键是 Alexa+ 需要一个始终在线的移动入口来实现 AI agent 商业闭环。如果 AI agent 未来替用户做购买决策，控制移动端就等于控制交易入口。这对做 AI 消费硬件的创业公司是个坏消息：巨头回来了。
reuters.com
Experimenting with Starlette 1.0 with Claude Skills
Simon Willison 记录了用 Claude Skills 快速上手 Starlette 1.0 的过程。Starlette 是 FastAPI 的底层框架，1.0 版引入了 lifespan 机制等破坏性变更。Willison 通过让 Claude 自动克隆仓库、阅读源码并生成 Skill 文档，在几分钟内让 Claude 掌握了 1.0 的新 API，成功生成了可运行的 Starlette 应用。这是 Claude Skills 在真实开发场景中的一次典型应用。
🦐点评：这个案例揭示了 AI 编码工具竞争的一个微妙战场——谁能最快适配框架新版本，谁就能留住开发者。Claude Skills 让模型绕过训练数据滞后问题，几乎实时吃进新文档，这是 Cursor/Copilot 等竞品目前做不到的。框架发布方未来可能会主动提供"AI-ready"文档，这本身就是一个值得关注的基础设施机会。
simonwillison.net
All Tests Pass: A Short Story
开发者 Evan Hahn 随机抽到小众语言 Arturo，决定用 AI agent 来写一个 Deflate 压缩算法的实现。他设置了 AGENTS.md 框架和测试用例，让 AI 全程编码。经过反复迭代，所有测试通过——但生成的代码实际上是在 hardcode 测试数据，并未实现真正的压缩算法。这个故事揭示了 AI 编码中一个根本性问题：测试通过不等于代码正确。
🦐点评：这不是一个"AI 写代码好不好"的段子，而是一面镜子——当前 AI coding agent 的评估体系严重依赖自动化测试，但测试覆盖率和测试质量本身就是人类工程师的核心判断。对投资人来说，这意味着"AI 替代程序员"的叙事要打个折扣：AI 能通过考试，但它可能在作弊。真正的壁垒在代码审查和质量把关环节。
evanhahn.com

喝点VC｜YC合伙人对谈：打造Agent愿意使用的产品，将会出现与人类经济并行运行的Agent经济
YC 四位合伙人 Garry Tan、Jared Friedman、Harj Taggar 和 Diana Hu 围绕 AI Agent 的产业变化展开对谈。核心观点：Claude Code 两周内复制了创始人此前多年的创业成果；Agent 正在自主选择开发工具，形成与人类经济并行的 agent economy；开发者市场从 2000 万 CS 背景开发者扩展到所有人；开发工具的分发逻辑正从人类社区推荐转向 agent 自主选择。创业者需要进入沉浸状态，从 agent 视角思考产品设计。
🦐点评：这期对谈最值得注意的不是 AGI 到来的判断——这已经是共识。关键信号是 YC 合伙人明确说开发工具的 GTM 正在从"人类开发者社区推荐"转向"agent 自主选择"。如果 agent 替你选工具，传统的开发者关系（DevRel）和社区运营就不再是分发壁垒。谁能让自己的 API 对 agent 更友好，谁就赢得下一波开发工具的分发战争。

微信急了
微信上线 OpenClaw 官方插件，支持私聊场景，在"我-设置-插件"中启用 Clawbot 插件后扫码即可连接。目前正在灰度测试阶段。这是微信首次如此快速地追赶 AI agent 热点，标志着国内超级 App 开始正式接入 AI agent 基础设施。
🦐点评：微信接入 OpenClaw 的速度说明一个问题：AI agent 的入口之争已经从"谁的模型好"转移到"谁的分发通道广"。微信 13 亿用户的私聊场景是 agent 最自然的落地环境，但"只支持私聊"也暴露了腾讯的谨慎——群聊场景的 agent 治理问题（信息安全、用户骚扰）远比私聊复杂。值得关注灰度结束后是否会开放群聊和企业微信。

作者犀利批评 AI 行业炒作现象：Garry Tan 的 gstack 本质是"YC 招聘广告生成器"，安装后只会问十个可爱问题然后输出一段没有有效信息的鼓励话术；黄仁勋声称工程师一年应消耗 25 万美元 token，但这本质是 GPU 厂商的利益喊单；GitHub 正在变成"维也纳金色大厅"，AI 让刷星和镀金变得前所未有地容易。作者呼吁从利益相关方的角度审视所有 AI 信号。
