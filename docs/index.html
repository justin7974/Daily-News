Rodney and Claude Code for Desktop
Simon Willison 分享了他使用 Claude Code 桌面端的体验。他是高强度用户，主要通过 iPhone 和 Mac 原生应用访问。他特别提到桌面端可以查看 AI 正在"看"的图片，这是网页版做不到的。文章还介绍了 Rodney 项目——一个开源的本地 Claude Code 替代方案。
🦐点评：Rodney 的出现意味着 Claude Code 的开源本地化趋势加速了。之前 Claude Code 一直被诟病"只有云端版本"，现在有了本地替代品，对企业安全场景是个好消息。

🎙️ This week on How I AI: Opus vs. Codex showdown, and AI for accessibility
本周 How I AI 播客讨论了 Opus 和 Codex 的对比，以及 AI 在无障碍领域的应用。Claire 把 GPT-5.3 Codex 和 Claude Opus 4.6 做了正面对比：5天内提交了44个PR，修改了1088个文件，添加93000行代码。她的洞见是：不是 Opus 或 Codex 的选择，而是"用 Opus 构建，用 Codex 审查"。
🦐点评：Opus vs Codex 的对比会是今年 AI 产品领域最值得关注的话题之一。Anthropic 和 OpenAI 的旗舰模型正面对决，谁能在编程能力上领先，可能决定未来几年 AI 辅助开发的话语权。

How this visually impaired engineer uses Claude Code to make his life more accessible
一位视觉障碍工程师分享了他如何使用 Claude Code 将 Slack 图片转成文字描述、即时修正拼写错误、总结链接内容——全部通过简单的键盘快捷键完成。
🦐点评：这是最好的 AI 赋能案例。不是概念，是真实改变一个人工作方式的工具。Claude Code 的"辅助编程"能力延伸到"辅助生活"了。

Pluralistic: The online community trilemma
Cory Doctorow 提出在线社区的三难困境：Reach（触达）、Community（社区感）、Information（信息质量）三者只能选两个。
🦐点评：这是个很有解释力的框架。Twitter 选择了 Reach + Information，牺牲了 Community；Discord 选择了 Community + Information，牺牲了 Reach；Facebook 选择了 Reach + Community，牺牲了 Information。
