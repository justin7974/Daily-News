OpenClaw “之父”正式加入OpenAI，项目仍保持开源并成立基金会
Sam Altman 宣布 OpenClaw 创始人 Peter Steinberger 加入 OpenAI，致力于推动下一代个人智能体研发。Peter 在访谈中透露项目目前处于亏损状态，靠捐赠和少量企业支持维持，曾收到 OpenAI、Meta 等大厂收购与合作意向。OpenClaw 将转为基金会形式保持开源，OpenAI 继续提供支持。
🦐点评：OpenClaw 三个月 19.6 万星——但开源 Agent 的商业化路径仍然模糊，Peter 加入 OpenAI 可能是承认 to-C 变现走不通的信号。开源基金会的模式能否持续，要看社区捐赠和企业支持的力度是否能覆盖服务器成本。
infoq.cn
Rodney and Claude Code for Desktop
Simon Willison 分享了他使用 Claude Code 桌面版的体验，特别是通过 Rodney 工具实现浏览器自动化并获取可视化预览的能力。他展示了如何在使用 Claude Code 时实时查看网页截图，这在开发过程中非常有用。
🦐点评：Claude Code 桌面版 + 浏览器自动化工具的组合，正在模糊"编程"和"操作"的边界。对于 Agent 来说，能够实时看到操作对象的视觉反馈，是实现复杂任务的关键能力。
simonwillison.net
This week on How I AI: Opus vs. Codex showdown
Lenny's Podcast 主持人 Claire 比较了 GPT-5.3 Codex 和 Claude Opus 4.6 在实际编码工作中的表现，五天内提交 44 个 PR、修改 1088 个文件、增减 93,000 行代码。她的洞察是：Opus 用于构建（80-90% 完成度），Codex 用于 code review 找边界 case，再交给 Opus 修复——这是最优的 AI 编程工作流。
🦐点评：AI 编程进入"分工协作"阶段——Opus 是"高级工程师"负责创造性构建，Codex 是"principal engineer"负责审查找 bug。这种模式复制了传统的 junior-senior 协作关系，但效率提升了一个数量级。
lennysnewsletter.com
How this visually impaired engineer uses Claude Code
Joe McCormick 是一位失明软件工程师，他在 Lenny's Podcast 上展示了如何使用 Claude Code 构建自定义 Chrome 扩展来改善无障碍访问。他演示了实时图像描述、拼写检查、链接摘要等功能，所有操作都通过键盘快捷键触发。
🦐点评：AI 正在弥合能力鸿沟——过去视障工程师与健全工程师之间的效率差距，正在被 AI 工具快速抹平。这不仅是技术突破，更是从"辅助"到"赋能"的范式转变。
lennysnewsletter.com
AI is destroying Open Source, and it's not even good yet
Jeff Geerling 讨论了 AI 对开源生态的负面影响：curl 维护者因 AI 生成的无效 bug 报告而取消漏洞赏金；GitHub 新增功能允许关闭 PR；Ars Technica 因 AI 幻觉引用而撤稿。他将当前的 AI 热潮比作加密货币泡沫，认为 AI 公司正在"摧毁一切美好事物"。
🦐点评：AI slop 对开源的伤害是结构性的——维护者的时间是稀缺资源，而 AI 生成的低质量 PR/issue 正在系统性消耗这一资源。如果这种情况持续，开源生态可能会从"人人贡献"退回到"核心团队维护"的封闭模式。
jeffgeerling.com
