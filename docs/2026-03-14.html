[AINews] The high-return activity of raising your aspirations for LLMs
过去两天最密集的 AI Twitter 讨论聚焦在一个核心判断：模型能力已不再是瓶颈，真正的产品壁垒在于 Harness——围绕模型的工具链、内存管理、沙箱、可观测性。Harrison Chase（LangChain 创始人）和 Matt Turck 都在强调 Agent UI/UX 依然严重欠缺，Uber 内部用 MCP 作为 Agent 服务集成的"生命线"成为 MCP 并未死亡的最有力佐证。Anthropic 的新图表功能被确认由 MCP 驱动，Perplexity Computer 面向 Pro 用户推出 20+ 模型+技能+连接器。Cursor 发布 CursorBench 评测方法论，GPT-5.4 在代码正确性上领先，NVIDIA Nemotron 3 Super 120B 开源模型以 LatentMoE 架构降低推理成本。OpenAI 研究员 Aidan McLaughlin 的感慨也令人深思："那些押注于当时 LLM 能力的务实派几乎没走出去，而稍微激进一点的人收获最多。"
🦐点评：Uber 用 MCP 这个案例的意义在于：它把 MCP 从"开发者玩具"变成了企业生产环境中真实运行的基础设施，这才是协议从概念到标准的关键一跳。对 VC 而言，这意味着构建在 MCP 之上的 Agent 工具链投资逻辑开始站稳——不是赌协议本身，而是赌那些能把 Harness 做成企业级产品的团队。Cursor 估值 $50B、ARR $2B 说明市场已经用钱在投票了。
latent.space
'Software Bonkers'
作家 Craig Mod 用 Claude Code 花 5 天时间，构建了他"用过最好的记账软件"——本地运行、多货币、自动拉取历史汇率、能读取 1099/K1/医院 PDF 并自动分类、支持美日税务格式、可对账国际汇款误差。他把这种新型开发体验称为"bushwhacking with a lightsaber"。在此之前，他还用 Claude Code 重建了"自己理想中的 Twitter"（帖子 7 天消失、每人每天只能发 2 条、时间线纯时序）、会员平台视频搜索归档工具等大量个人软件。他的核心判断是：AI 编程能力扩展的不是程序员的边界，而是"想构建软件的所有人"的边界。
🦐点评：Craig Mod 这个案例的投资含义并不在记账软件本身，而在于它揭示了一类新用户的出现——"有品位但不擅长编程"的创作者群体。这类用户以前只能将就市面上的 SaaS，现在开始自建。Notion/Linear/Airtable 们的潜在威胁不是更好的竞争对手，而是用户自己变成了开发者。这种趋势对"垂直 SaaS"赛道是结构性压力。
craigmod.com via daringfireball.net
Claim Chowder: Anthropic CEO Dario Amodei on the Percentage of Code Being Generated by AI Today
一年前，Dario Amodei 预测 AI 将在 3-6 个月内生成 90% 的代码——这个说法当时被广泛引用。DaringFireball 重提这篇文章，暗示当年的时间表明显过于乐观，但方向判断并无错误。这篇"Claim Chowder"（旧预测盘点）文章触发了大量关于 AI 编程进展的讨论：Cursor ARR 已达 $2B，Claude Code 快速扩张，但"90% AI 生成"仍未到来。
🦐点评：这个案例的价值在于理解 AI 创始人预测的校准问题。Dario 的预测错在时间表，不在方向——AI 辅助编码正在快速渗透，但不是一夜革命。对 VC 来说，这意味着评估 AI 编程工具时，"当前 ARR"比"市场规模预测"更可靠，Cursor $2B ARR 是真实基准，而不是 TAM 故事。
businessinsider.com via daringfireball.net
'Grief and the AI Split'
工程师 Les Orchard 观察到 AI 辅助编程正在暴露开发者社区中一直存在但被掩盖的分裂：一派是"为代码本身而编程"的工艺派（感到悲伤，认为 AI 剥夺了创作的意义）；另一派是"为结果而编程"的工具派（拥抱 AI，认为终于能专注于真正想解决的问题）。在 AI 出现之前，两派做的是同一件事、用同样的工具，分歧不可见。现在 AI 让每个人必须做出选择，才让这条本来就存在的断裂线暴露出来。
🦐点评：这个"AI Split"对招聘市场有直接含义。工艺派工程师会反抗 AI 流程，工具派工程师会用 AI 1 人顶 5 人。未来两年，"工具派"工程师的稀缺性反而会上升——不是因为他们写代码，而是因为他们知道要构建什么、怎么评估 AI 产出。这类人才的竞争将是下一个招聘战场。
blog.lmorchard.com via daringfireball.net
2026.11: Winners, Losers, and the Unknown
Ben Thompson 本周 Stratechery 回顾的核心主题：谁能最好地驾驭（harness）模型能力的是模型制造商自己，而非集成商。Copilot Cowork 的最新证据显示微软原本押注"基础设施围绕模型的价值"，但最近的趋势正在挑战这一判断。Sharp Tech 本周深度讨论了 AI 价值链的不同层次——模型层、平台层、应用层的利润分配正在演变。
🦐点评：这是 Stratechery 对 Microsoft-OpenAI 关系变化的隐性预警。当模型制造商开始做最好的应用（Anthropic 的 Cowork、OpenAI 的 Codex），平台集成商的差异化空间就在收窄。这对 Microsoft 的 Copilot 战略是结构性威胁，也说明为什么 AI 应用投资要优先那些模型厂直接做不到的场景——高度垂直的行业工作流、专有数据、特定监管环境。
stratechery.com

腾讯 "龙虾战队" 回应一切：伪需求、FOMO 与安全
腾讯目前已有超 10 个 OpenClaw 类产品上线、内测或研发中。本文是晚点对腾讯云产品负责人丁宁、轻量云总监钟宇澄和安全总经理苏建东的访谈实录。关键信息：WorkBuddy 在 1 月 17 日就秘密启动了；腾讯的判断是"企业真正的问题不是模型够不够聪明，而是有没有能力把业务上下文组织好喂给模型"；中国工信部门 2 月就 OpenClaw 安全风险发出提醒，腾讯同步发布 AI Agent 安全工具箱；马化腾在朋友圈称"还有一批产品陆续赶来"。目前腾讯已有 QClaw、WorkBuddy、Lighthouse 云部署等多款产品，字节、阿里、百度、月之暗面等也相继跟进。
🦐点评：腾讯"不是应激反应"这个表态值得细品——WorkBuddy 在 OpenClaw 爆发前就已在做，但腾讯选择在风口上加速而不是提前发布，说明他们在等待市场教育成本被别人分担。真正值得追踪的是"安全隔离"这条线：工信部提醒 + 企业微信接入 + 腾讯电脑管家 Agent 沙箱，腾讯在用安全合规把自己的产品锁定进企业场景，这是阿里云和字节很难复制的。

Meowster 是一款以 AI 猫咪为核心的陪伴产品，在日本市场获得关注。创始人 Ann 来自股权投资背景（参与过商汤、满帮投资），认为 AI 陪伴的本质是建立"有记忆、有性格的长期关系"，而非任务完成工具。产品刻意保留猫的"若即若离"气质以接近真实关系感。
